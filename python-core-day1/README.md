# DAY 1 — PYTHON CORE (RUTHLESS VERSION)

This is NOT DSA grinding.
This is Python-for-Data-Science thinking.

You will learn:

How to think in data

How to write reusable logic

How not to embarrass yourself in interviews

WHAT YOU ARE LEARNING TODAY (NO FLUFF)

By end of Day 1, you must be able to:

Write pure functions

Manipulate lists, dicts, tuples

Use loops vs comprehensions correctly

Handle edge cases

Think about time & space at a basic level

No trees. No graphs. No LeetCode ego-stroking.

THE 20 PROBLEMS (DO ALL — NO SKIPPING)
CATEGORY 1: LISTS & LOOPS (Problems 1–5)

These expose lazy thinking.

Reverse a list without using reverse()

Remove duplicates from a list (preserve order)

Find second largest element

Rotate list by k positions

Split list into chunks of size n

👉 Learn:

Indexing

Loop control

Boundary conditions

CATEGORY 2: STRINGS (Problems 6–9)

Interviewers love these because people mess them up.

Check if a string is a palindrome (ignore case & spaces)

Count frequency of characters

Find first non-repeating character

Reverse words in a sentence

👉 Learn:

String immutability

Dict-based counting

Clean logic

CATEGORY 3: DICTIONARIES & COUNTERS (Problems 10–13)

This is real DS logic.

Count occurrences in a list

Group words by length

Merge two dictionaries (sum values if keys match)

Find key with max value (without using max())

👉 Learn:

Hash maps

Frequency analysis

Memory vs speed tradeoff

CATEGORY 4: FUNCTIONS & CLEAN CODE (Problems 14–16)

This is where juniors fail.

Write a function with default arguments

Write a function that returns multiple values

Refactor a long script into 3 clean functions

👉 Learn:

Reusability

Separation of concerns

Readability > cleverness

CATEGORY 5: BASIC COMPLEXITY THINKING (Problems 17–20)

No math. Just honesty.

Detect duplicates in a list (optimize it)

Find common elements between two lists

Find missing number in a sequence

Count pairs with a given sum

👉 Learn:

O(n) vs O(n²)

When to use sets

Interview explanation skill

HOW TO SOLVE (NON-NEGOTIABLE METHOD)

For every problem:

Write brute-force solution

Write optimized solution

Add edge cases

Write a short explanation

If you skip step 2 or 4 → you failed the task.
